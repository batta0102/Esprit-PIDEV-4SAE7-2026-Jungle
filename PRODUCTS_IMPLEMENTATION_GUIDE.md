## 🔧 Key Features

### 1. **Product Interface**
```typescript
interface Product {
  idProduct?: number;      // Auto-generated by backend
  name: string;            // Required
  category: string;        // Required
  description: string;     // Required
  image?: string;          // Optional URL
  price?: number;          // Optional
  stock: number;           // Required
}
```

### 2. **ProductService** (Dependency Injection)
- ✅ GET `/allProducts` - Load all products
- ✅ GET `/getProduct/{id}` - Load single product
- ✅ POST `/addProduct` - Create new product
- ✅ PUT `/updateProduct/{id}` - Update existing product
- ✅ DELETE `/deleteProduct/{id}` - Delete product
- ✅ Comprehensive error handling with logging

### 3. **Component State (Angular Signals)**
```typescript
products = signal<Product[]>([]);          // All products
loading = signal(false);                   // Loading state
error = signal<string | null>(null);       // Error messages
showForm = signal(false);                  // Form visibility
isEditMode = signal(false);                // Add vs Edit mode
currentProduct = signal<Product>({...});   // Form data
```

### 4. **Core Methods**

#### Load Products
```typescript
loadProducts(): void
// Fetches all products from API
// Handles field transformation (id → idProduct)
// Updates products signal
```

#### Add Product
```typescript
openAddForm(): void
saveProduct(): void
// POST request to /addProduct
// Removes idProduct for new products
// Reloads list on success
```

#### Update Product
```typescript
openEditForm(product: Product): void
saveProduct(): void
// PUT request to /updateProduct/{idProduct}
// Passes full product object with ID
// Reloads list on success
```

#### Delete Product
```typescript
deleteProduct(product: Product): void
// Confirmation dialog
// DELETE request to /deleteProduct/{idProduct}
// Reloads list on success
```

### 5. **TrackBy Function** (NG0955 Prevention)
```typescript
trackByIdProduct(index: number, product: Product): number | undefined {
  return product.idProduct || index;
}

// Usage in template:
// @for (product of products(); track trackByIdProduct($index, product))
```

### 6. **Form Management**
- **Toggle:** `showForm` signal controls visibility
- **Mode:** `isEditMode` signal indicates Add vs Edit
- **Data:** `currentProduct` signal holds form values
- **Validation:** Required fields checked before submit
- **Reset:** `closeForm()` clears all form state

## 🐛 Common Issues & Solutions

### Issue 1: "Connection Refused"
```
❌ Error: net::ERR_CONNECTION_REFUSED at http://localhost:8085
```
**Solution:** Start API Gateway on port 8085
```bash
cd api-gateway
mvn spring-boot:run
```

### Issue 2: "CORS Error"
```
❌ Error: No 'Access-Control-Allow-Origin' header
```
**Solution:** Add CORS config to Spring Boot (see CORS_FIX_GUIDE.md)

### Issue 3: "NG0955 duplicate keys"
```
❌ Error: NG0955: Duplicate key in *ngFor
```
**Solution:** Already handled with `trackByIdProduct()` function

### Issue 4: "404 Not Found"
```
❌ Error: 404 on PUT /updateProduct/2
```
**Solution:** Verify endpoint URLs match backend
- Check baseUrl: `http://localhost:8085/products`
- Check endpoint paths match backend

### Issue 5: "idProduct is undefined"
```
❌ Error: Can't read property 'idProduct' of undefined
```
**Solution:** Backend might return `id` instead of `idProduct`
- Service handles this in `loadProducts()` with transformation
- Or ensure backend uses `idProduct` field name
